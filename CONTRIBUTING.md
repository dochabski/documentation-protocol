# Contributing

This protocol is in draft form until pilot review is complete.

## Change Process

1. Identify the affected protocol product in `specs/product-catalog.yaml`.
2. Propose the change with rationale, affected files, conformance impact, and evidence.
3. Update YAML source files before updating rendered Markdown.
4. Update related templates, checklists, schemas, or documentation when requirements change.
5. Preserve review and decision records for release-affecting changes.

## Required Review

Changes to product specs, templates, schemas, quality gates, release rules, or governance rules require review before public release.

## Current Draft Limits

The license, public maintainer list, and first pilot artifact have not been selected yet.
